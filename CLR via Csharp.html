    CLR 基础

    CLR执行模型

    CLR:公共语言运行时(Common Language Runtime)，是可以供多种语言使用的运行时。ClR其实不关心具体的哪种语言的源代码，因为其最终面向的是由托管模块合并的程序集，是经过编译器编译的代码。 CLR功能:内存管理,程序集加载,安全性,异常处理,线程同步

    应用程序如何执行。

    C#源代码 ->经编译器编译成托管模块(中间语言和元数据)(managed module) ->托管模块都需要CLR才能运行。(托管模块包括PE32或PE32+(可移植执行体)，CLR头，元数据，IL) ->同样使用编译器将托管模块合并成程序集(CLR实际操作的是程序集，而不是托管模块) ->加载程序集代码 ->执行程序集的代码(JIT把IL转换成CPU指令) CLR检测Main代码引用的所有类型(通过内部数据结构访问). 类型定义的每个方法都有一个对应的记录项(引用,地址,指针).根据地址,找到方法的实现 对Console类型的对象进行初始化的时候,生成JITCompiler(未编档的函数) 当首次调用的时候,JITCompiler才将对应的方法的IL代码编译成本机的CPU指令.

    IL(中间语言,托管代码，面向对象机器语言):编译器编译源代码时生成的代码.CLR将IL编译成本机CPU指令 元数据：一个数据表集合(类型和成员)，一个是模块引用的数据(导入的类型和成员)，嵌入到Dll或者是exe中

    程序集(CLR里，程序集相当于组件，可以是Dll，也可以是exe)：一个或多个模块/资源文件的逻辑性分组。 程序集是重用，安全性，以及版本控制的最小单元。

    JIT编译器(just in time即时):CLR里面的编译器(即是CLR的一个组件)，把方法(函数)的IL转换成本机的CPU指令。

    首次调用方法： Main方法执行之前，CLR先检查Main方法里面的所有类型，如果没有该类型，则先创建并分配该类型对象。

    CLR如何创建类型对象，对象，以及阐述这些对象里面静态方法 实例方法 虚方法的调用区别

    注意：类也是对象，一切皆对象 对于CLR来说，创建对象之前，需要知道对象属于什么类。所以先要创建类这个对象(类对象区别类的对象)，也就是描述这个对象。然后创建对象时候将类型对象指针指向该类型对象。 CLR将某个方法转的IL转换成CPU指令的时候，会检查确定方法里面的类型是否已经创建，如果没有创建则创建 静态方法(类的方法)：CLR找到对应的类型对象，JIT编译器然后在类型对象的方法表中找方法对应的纪录项，对方法进行JIT编译(如果需要的话)，然后再调用执行 (变量类型的类型对象方法) 非虚实例方法：JIT编译器找到调用该方法变量的类型对应的类型对象 (变量类型的类型的对象的方法) 虚实例方法：JIT编译器先根据变量，并跟随地址找到对象，再跟随类型对象指针找到类型对象，从而找到对象的类型对象的方法。(变量对象的类型对象指针所对应的类型对象的方法)(类型的对象的类型对象里面的方法)

    堆上所有对象包含两个额外成员:类型对象指针和同步块索引

    readonly

    综上所述，可以得出结论：readonly修饰的字段，其初始化仅是固定了其引用（地址不能修改），但它引用的对象的属性是可以更改的。

    当然，这里的“地址不能修改”指的是在代码中不能再对readonly变量进行再赋值，实际运行情况中可能会遇到GC或反射改变内存的情况。但是在本文所说的环境下不必考虑。（2013-9-27 14：30修改）

    所以，大胆放心的用readonly吧，这样还可以防止不必要的再赋值，保证了这个对象的安全性。

    const 一个包含不能修改的值的变量。 readonly 允许把一个字段设置成常量，但可以执行一些运算，可以确定它的初始值。 readonly 是实例成员，所以不同的实例可以有不同的常量值，这使readonly更灵活。 readonly 字段可以在声明或构造函数中初始化。因此，根据所使用的构造函数，readonly 字段可能具有不同的值。 2. const 字段是编译时常数，而 readonly 字段可用于运行时常数。 3. const 默认就是静态的，而 readonly 如果设置成静态的就必须显示声明。 4．const 对于引用类型的常数，可能的值只能是 string 和 null。 readonly可以是任何类型 总结 const只能在初期就使用常量初始化好。对于每一次编译后的结果，const的值是固定的，而readonly的值是可以在运行的时候才确定值的~~

    如果主程序编译时加载另外一个DLL程序集的CONST变量,如果更改了变量,那么得重新主程序和DLL同时编译.因为CONST是在编译的时候写进去的.如果需要只编译DLL程序集,把CONST改为readonly
