iptables的规则 简单来说，iptables防火墙是由一系列的规则(rule)组成，一个数据请求进来，会依次和这些规则进行比较，如果正好符合规则的定义，那这个数据请求要么会被接收ACCEPT，要么被拒绝DRIP。如果不符合任何规则的定义，最后缺省的规则会被应用。

Iptables [-ttable] COMMAND chain CRETIRIA -j ACTION

分为三种类型 -ttable有三种选项：filter，nat 和managle

vi /etc/sysconfig/iptables *filter #过滤器 #filter包括input forward output三种类型

#input：输入 :INPUT ACCEPT [0:0]

#output：输出 :OUTPUT ACCEPT [0:0]

#forward：转换 :FORWARD ACCEPT[0:0]

-A FORWARD -i eth1 -o eth1 -j ACCEPT

*nat #network address translation #同filter表一样，nat表也有三条缺省的"链"(chains)

:PREROUTING ACCEPT[0:0] :POSTROUTING ACCEPT[0:0] :OUTPUT ACCEPT[0:0] -A POSTROUTING -s 192.168.8.0/24 -d 172.18.100.104 -o eth0 -j SNAT -to--source 172.18.100.1

-j SNAT：源网络地址转换，SNAT就是重写包的源IP地址(代理) 系统先路由——>再过滤（FORWARD)——>最后才进行POSTROUTING SNAT地址翻译
