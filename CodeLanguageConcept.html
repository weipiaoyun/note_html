REST API

REpresentational (Representation of those resources) State Transfer

  1. Architecture style for designing networked application.
  2. Relies on a stateless, client-server protocol,always HTTP
  3. treats server objects as resource that can be created or destroyed.
  4. can be used by virtually any programming language.

Rest API and restful API are essentially the same thing
