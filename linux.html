setenv是csh的命令; 在bash中是没有setenv的

csh/tcsh有两种类型的变量:局部变量和环境变量.局域变量是在shell内部声明的;环境变量是全局域的变量. 使用set来定义局部变量, 使用setenv可以定义环境变量; 局部变量只对本shell有效, 不能传递给子shell; 但环境变量可传递给子shell. setenv有点类似于bash中export一个变量.

当然, set 和setenv的语法不一样

  1. set varname = value;
  2. setenv varname2 valu2;

set语法set name=value setenv语法setenv name value 最大的区别是，set变量只对当前进程有效，不会传递给子进程 setenv变量不仅对当前进程有效，而且会传递给子进程 而当同时使用set和setenv设置一个变量时，当前进程中，会优先使用set设置的值

在Bash中也有类似的区别 BASH不需要显示声明变量使用=号直接能声明并初始化变量，但是如果没有export，是不能被子进程使用的。

其他局域网电脑连接自己电脑上的虚拟机

1，把VM的网络适配器由NAT模式改为桥接模式，详情看计算机网络
2，修改配置文件/etc/sysconfig/network-scripts。改为如下：

  1. Broadcom Corporation NetXtreme BCM5764M Gigabit Ethernet PCIe

DEVICE=eth1 BOOTPROTO=none BROADCAST=192.168.8.255 HWADDR=e8:39:35:30:cf:d4 IPADDR=192.168.8.94 NETMASK=255.255.255.0 NETWORK=192.168.8.0 ONBOOT=yes GATEWAY=192.168.8.1 TYPE=Ethernet

或者是：

  1. Broadcom Corporation NetXtreme BCM5764M Gigabit Ethernet PCIe

DEVICE=eth0 BOOTPROTO=none BROADCAST=172.18.255.255 HWADDR=E8:39:35:30:CF:D5 IPADDR=172.18.1.42 IPV6INIT=yes IPV6_AUTOCONF=yes NETMASK=255.255.0.0 NETWORK=172.18.0.0 ONBOOT=yes TYPE=Ethernet

3，重启网络。即可。 如果不行，手动设置网络。ifconfig 192.168.8.94 netmask 255.255.255.0 up
