1，什么是索引 比如如果一部字典，查找一个字，如果没有索引，要一页一页地翻，直到找到该字为止。如果加上索引，比如按偏旁算法作为索引，或按拼音算法作为索引，很快就可以找到字，然后可以找到所对应的页(指针)。很快就可以找到该字。这就是字典的索引。数据库的索引是同样的意思。

2，索引的特点 因为有了索引，查询速度快了。 因为有了索引，字典的厚度增加了，需要更多的纸张。数据库的物理空间增加了。需要更多的存储空间。 增加索引后，每次添加一个字，删除一个字，修改一个字都要重新更新索引。所以对于查询来说速度是快了，但对于修改，添加，删除来说速度慢了。 索引的更新对于数据库来说是自动更新的。 索引可以根据不同的算法对某一个或多个列进行按某种算法排序。然后对应有指针。查找速度块。

3，索引的分类 普通索引。 唯一索引。 主键索引。

创建索引的原则 1．选择唯一性索引 2．为经常需要排序、分组和联合操作的字段建立索引 3．为常作为查询条件的字段建立索引 4．限制索引的数目 5．尽量使用数据量少的索引 6．尽量使用前缀来索引 7．删除不再使用或者很少使用的索引

SQL Server 会有以下方法来查找您需要的数据记录：

【Table Scan】：遍历整个表，查找所有匹配的记录行。这个操作将会一行一行的检查，当然，效率也是最差的。

【Index Scan】：根据索引，从表中过滤出来一部分记录，再查找所有匹配的记录行，显然比第一种方式的查找范围要小，因此比【Table Scan】要快。

【Index Seek】：根据索引，定位（获取）记录的存放位置，然后取得记录，因此，比起前二种方式会更快。

【Clustered Index Scan】：和【Table Scan】一样。注意：不要以为这里有个Index，就认为不一样了。 其实它的意思是说：按聚集索引来逐行扫描每一行记录，因为记录就是按聚集索引来顺序存放的。 而【Table Scan】只是说：要扫描的表没有聚集索引而已，因此这二个操作本质上也是一样的。

【Clustered Index Seek】：直接根据聚集索引获取记录，最快！
