A Razor Page is very similar to ASP.NET MVC’s view component. It has basically same syntax and functionality as MVC.

The key difference between Razor pages and MVC is that
the model and controller code is also included within the Razor Page itself.

在新建项目里面新加一个model

  1. 在RazorPagesMovie解决方案下面创建Models文件夹。并在下面创建一个Movie这个model。
  2. 在Pages下创建Movies文件夹。在下面右击添加[新搭建机架的项目]。然后在通用->Razor页面里面选择[使用实体框架生成Razor页面(CRUD)]。点击添加。选在刚创建的movie model。数据上下文点击加号。默认。添加。

自动添加的文件

  • Pages/Movie下面自动添加了CDDEI开头的五个html文件。
  • 解决方案下新增了Data文件夹。并增加了RazorPagesMovieContext类。
  • 增加了appsettings.json配置文件，里面可以配置数据库。
  • 在startup里面的ConfigureServices里面增加了service.AddDbContext，增加访问数据库的上下文。
  • 如果想看数据库和表内容，切换到sql server。更改appsettings.json;Data Source=(local);uid=sa;pwd=123456;DataBase=RazorPagesTest1"

Examine the project files

Pages folder

Contains Razor pages and supporting files. Each Razor page is a pair of files:

  • .cshtml file that contains HTML markup with C# code using Razor syntax.
  • .cshtml.cs file that contains C# code that handles page events.

Supporting files have names that begin with an underscore For example, the _Layout.cshtml file configures UI elements common to all pages.This file sets up the navigation menu at the top of the page and the copyright notice at the bottom of the page.

wwwroot folder

Contains static files, such as HTML files, JavaScript files, and CSS files. For more information, see Static files in ASP.NET Core.For more information, see Static files in ASP.NET Core.

appSetting.json

Contains configuration data, such as connection strings.see Configuration in ASP.NET Core.

Program.cs

Contains the entry point for the program. For more information, see .NET Generic Host.

Startup.cs

Contains code that configures app behavior. For more information, see App startup in ASP.NET Core.
